## 工具选择与对比

### 现有工具 vs CameraCalibration 项目

| 特性       | calibrate_fisheye_180.py | CameraCalibration.IntrinsicCalib | 推荐场景                                       |
| ---------- | ------------------------ | -------------------------------- | ---------------------------------------------- |
| 工作模式   | 离线批量                 | 在线/离线任选                    | ✅ CameraCalibration 更灵活                    |
| 输入方式   | 本地图像文件             | 相机/视频/图像任选               | ✅ CameraCalibration 更方便                    |
| 实时反馈   | 无（只有最终报告）       | ✅ 实时重投影误差、去畸变预览    | ✅ CameraCalibration 体验更好                  |
| 参数灵活性 | 固定棋盘格(10×7)        | ✅ 完全可配置棋盘格、采样间隔等  | ✅ CameraCalibration 更灵活（支持你的 10×10） |
| 输出格式   | JSON                     | CalibData 对象（可自定义导出）   | ✅ 需转换步骤                                  |
| 代码可读性 | 文档完整                 | 代码结构化、类设计清晰           | 平分                                           |

**建议方案**：

- **方案1（快速）**：用 CameraCalibration 在线实时标定，快速获得初步结果
- **方案2（精度）**：用 CameraCalibration 先采样存图，再用离线模式精细调参，最终导出 JSON

## 方案 1：在线实时标定（推荐快速）

适用于：快速获得标定结果、实时反馈

### 步骤 1.1：连接相机

```bash
# 确保相机已物理连接到 USB
# 验证
ls /dev/video0
# 输出存在则成功
```

### 步骤 1.2：启动在线标定

```bash
cd /home/r1/Slam/CameraCalibration

# 最简单的用法（注：需指定棋盘规格）
python3 IntrinsicCalibration/intrinsicCalib.py \
    -fw 1920 -fh 1080 \
    -bw 9 -bh 9 \
    -size 18

# 指定相机分辨率和棋盘格
python3 IntrinsicCalibration/intrinsicCalib.py \
    -fw 1920 -fh 1080 \
    -bw 9 -bh 9 \
    -size 18

# 如果是后置相机 (camera ID > 0)
python3 IntrinsicCalibration/intrinsicCalib.py \
    -fw 1920 -fh 1080 \
    -bw 9 -bh 9 \
    -size 18 \
    -id 0

# 更多参数说明见 IntrinsicCalibration/README.md

**你的棋盘规格**：10×10 方格 = 9×9 内角点，1.8cm 方格 → `-bw 9 -bh 9 -size 18`
```

**参数详解**：

| 参数            | 说明                 | 默认值  | 建议值(180°鱼眼) |
| --------------- | -------------------- | ------- | ----------------- |
| `-fw`         | 帧宽度 (pixels)      | 1280    | 1920              |
| `-fh`         | 帧高度 (pixels)      | 1024    | 1080              |
| `-bw`         | 棋盘宽度(内角点数)   | 7       | 9                 |
| `-bh`         | 棋盘高度(内角点数)   | 6       | 9                 |
| `-size`       | 方格边长(mm)         | 10      | 18                |
| `-type`       | 相机类型             | fisheye | fisheye           |
| `-mode`       | 采样模式             | auto    | auto              |
| `-num`        | 最少采样帧数         | 5       | 15                |
| `-delay`      | 采样时间间隔(帧)     | 12      | 8                 |
| `-store`      | 是否定期保存抓取图像 | False   | True              |
| `-store_path` | 保存路径             | ./data/ | ./calib_images/   |

### 步骤 1.3：交互操作

**启动后的窗口操作**：

```
┌─────────────────────────────────┐
│  raw_frame 窗口                 │
├─────────────────────────────────┤
│                                 │
│  [实时相机画面]                 │
│  对准棋盘格                      │
│                                 │
├─────────────────────────────────┤
│ 按键说明：                       │
│ [Space] - 开始标定              │
│ [r]     - 重置（丢弃所有采样）  │
│ [q]     - 退出                  │
│ [s]     - 保存当前帧             │
└─────────────────────────────────┘
```

**标定流程**：

1. **启动程序** → 看到 `raw_frame` 窗口
2. **按空格键** → 开始标定流程
3. **缓慢移动相机** → 从多个角度对准棋盘格

   - 前后移动（改变棋盘大小）
   - 左右移动（改变棋盘水平位置）
   - 旋转相机（改变棋盘角度）
   - 倾斜相机（改变棋盘竖直位置）
   - 覆盖棋盘在图像中的不同位置（中心、四角、边缘）
4. **采样要点**：

   - 每采集一帧，窗口会显示已采集帧数 (显示在窗口标题或打印输出)
   - 采集到 `-num` (默认5) 帧后，程序自动计算初步标定结果
   - 此后按下 `Space` 会继续采样并优化结果 (一般采集 15-25 帧)
   - 按 `r` 可重置重新开始
   - 按 `q` 退出
5. **观察反馈**：

   - 采集 5-10 帧后应该看到 `undistort_frame` 窗口出现
   - 显示去畸变后的图像（应看到棋盘变得更"正常"）
   - 重投影误差会持续改进（通常从 5.0 逐步降到 0.2-0.5）

### 步骤 1.4：查看结果


适用于：标定图像质量要求高、需要精细控制

### 步骤 2.1：采集标定图像

#### 方式 A：用在线模式存图（推荐）

```bash
cd /home/r1/Slam/CameraCalibration

# 运行在线模式，同时存图
```

#### 方式 B：手动视频录制

```bash
# 用 OpenCV 或其他工具录制视频
python3 << 'EOF'
import cv2

cap = cv2.VideoCapture(0)
cap.set(cv2.CAP_PROP_FRAME_WIDTH, 1920)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 1080)

out = cv2.VideoWriter('calibration_video.mp4', 
                      cv2.VideoWriter_fourcc(*'mp4v'), 
                      30, (1920, 1080))

print("Recording... Press 'q' to stop")
while True:
    ret, frame = cap.read()
    if not ret: break
  
    cv2.imshow('frame', frame)
    out.write(frame)
  
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
out.release()
cv2.destroyAllWindows()
print("Video saved: calibration_video.mp4")
EOF
```

### 步骤 2.3：手动筛选（可选但推荐）

在 `manual` 模式下，程序会逐帧显示：

```
[q] 跳过这一帧 (不加入标定)
[Space] 保留这一帧 (加入标定)
[r] 重置 (重新开始)
```

**筛选原则**：

- ✅ 棋盘格清晰、所有内角点都能看到
- ✅ 棋盘占据图像 30-80% 的区域
- ❌ 棋盘模糊、部分角点看不清
- ❌ 棋盘过小（< 20% 图像）或过大（> 90% 图像）
- ❌ 图像有明显运动模糊或过曝

## 标定质量评估

### 评估指标

#### 1. 重投影误差 (Reprojection Error)

```
输出示例：
Reprojection Error: 0.25 pixels

判断标准：
┌──────────────────────┬──────────┬────────────┐
│ 误差范围             │ 质量评级 │ 是否可用   │
├──────────────────────┼──────────┼────────────┤
│ < 0.3 pixels         │ 优秀     │ ✅ 可用   │
│ 0.3 - 0.5 pixels     │ 良好     │ ✅ 可用   │
│ 0.5 - 1.0 pixels     │ 可接受   │ ✅ 勉强   │
│ 1.0 - 2.0 pixels     │ 一般     │ ⚠️  需验证 │
│ > 2.0 pixels         │ 较差     │ ❌ 重新标定 │
└──────────────────────┴──────────┴────────────┘
```

#### 2. 畸变系数合理性

对于 180° 鱼眼：

```python
# 标定结果示例
K = [[595.3,   0.0, 960.0],
     [  0.0, 595.1, 540.0],
     [  0.0,   0.0,   1.0]]

D = [-0.1234, -0.0567, 0.0012, -0.0008]

# 合理性检查
检查项                        期望值                    实际值
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
fx ≈ fy                      绝对值相近 (误差<2%)      ✅ 595.3 ≈ 595.1
cx ≈ width/2                接近图像中心              ✅ 960 ≈ 1920/2
cy ≈ height/2                接近图像中心              ✅ 540 ≈ 1080/2
k1 (一阶畸变)                 负值 (内缩鱼眼)          ✅ -0.1234 (合理)
k2 (二阶)                     通常远小于 k1           ✅ -0.0567 << -0.1234
k3, k4 (高阶)                 通常接近 0              ✅ 0.0012, -0.0008
```

#### 3. 视觉验证

在标定界面中观察 `undistort_frame` 窗口：

```
好的标定结果：
- 棋盘格变得更"正常"（不太弯曲）
- 周边不会过度校正反弯
- 去畸变后图像边缘无明显黑边

差的标定结果：
- 棋盘仍然明显弯曲
- 周边过度校正（反向弯曲）
- 去畸变后边缘大量黑边
```

## 导出为 JSON 格式

CameraCalibration 的输出需要转换为 JSON 供 AR 项目使用。

### 自动导出脚本

创建 `export_calib_result.py`：

```python
#!/usr/bin/env python3
"""
将 CameraCalibration 的标定结果导出为 JSON 格式
兼容 ar_grid_detector 的参数格式
"""

import json
import numpy as np
import sys

# 这些值由 CameraCalibration 计算得到
# 通过脚本或手工从终端输出复制下来

CAMERA_MATRIX = np.array([
    [595.3,   0.0, 960.0],
    [  0.0, 595.1, 540.0],
    [  0.0,   0.0,   1.0]
])

DISTORTION_COEFFS = np.array([
    -0.1234,    # k1
    -0.0567,    # k2
     0.0012,    # k3
    -0.0008     # k4
])

IMAGE_WIDTH = 1920
IMAGE_HEIGHT = 1080

# 导出为 JSON
output = {
    "camera_matrix": {
        "fx": float(CAMERA_MATRIX[0, 0]),
        "fy": float(CAMERA_MATRIX[1, 1]),
        "cx": float(CAMERA_MATRIX[0, 2]),
        "cy": float(CAMERA_MATRIX[1, 2]),
        "data": CAMERA_MATRIX.flatten().tolist()
    },
    "distortion_coefficients": {
        "data": DISTORTION_COEFFS.flatten().tolist()
    },
    "image_width": IMAGE_WIDTH,
    "image_height": IMAGE_HEIGHT,
    "camera_model": "fisheye_kannala_brandt"
}

# 保存
with open("fisheye_calibration.json", "w") as f:
    json.dump(output, f, indent=2)

print("✓ 标定结果已导出：fisheye_calibration.json")
print(json.dumps(output, indent=2))
```

1. **在终端中运行标定** → 看到最终输出

## 常见问题排查

### Q1：棋盘格无法检测

**症状**：运行标定程序后看不到角点被检测

**原因**：

1. 棋盘太模糊或太小
2. 照明不足，对比度低
3. 棋盘角数设置错误

**解决**：

```bash
# 先用简单脚本测试棋盘检测
python3 << 'EOF'
import cv2
import numpy as np

cap = cv2.VideoCapture(0)
BOARD_W, BOARD_H = 11, 8  # 内角点数

while True:
    ret, frame = cap.read()
    if not ret: break
  
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    ok, corners = cv2.findChessboardCorners(
        gray, (BOARD_W, BOARD_H),
        flags=cv2.CALIB_CB_ADAPTIVE_THRESH | cv2.CALIB_CB_NORMALIZE_IMAGE
    )
  
    if ok:
        cv2.drawChessboardCorners(frame, (BOARD_W, BOARD_H), corners, ok)
        cv2.putText(frame, "DETECTED", (50, 50), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 0), 2)
    else:
        cv2.putText(frame, "NOT FOUND", (50, 50), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 0, 255), 2)
  
    cv2.imshow('test', frame)
    if cv2.waitKey(1) & 0xFF == ord('q'): break

cap.release()
cv2.destroyAllWindows()
EOF
```

```bash
    -delay 6         # 更频繁采样
```

### Q4：去畸变后图像黑边太多

**症状**：`undistort_frame` 窗口边缘有大量黑色区域

**原因**：程序设置的焦距缩放系数 (`-fs`) 太高

**症状**：九宫格在图像上显示位置不对或扭曲
